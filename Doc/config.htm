Configuration

The tagsets for the parts of speech and the dependency relations used in annotations can be configured according to the chosen corpus. This default corpus is chosen according to the language locale.

Corpus specific configuration is provided through resource files. The following properties can be specified:

corpus
the name of the corpus
language
the language used in the corpus
pos
the POS tags to use (one per line, paired with an optional comment to be displayed as a tooltip)
deprel
the set of dependency relations (one per line, paired with an optional comment to be displayed as a tooltip)
tagger
URL of an optional Part of Speech tagging service
The configuration file for each corpus is stored in the directory dga. Other corpora can be added by placing their configuration file in this directory. They must be named as DGA_name where name is the short name for the corpus, typically an ISO Language Code. DGA will load them at startup. A set of default configuration files is provided and associated to the locale name of their language. Here is the default configuration file for English dga/DGA_en.xml:


<?xml version="1.0" encoding="UTF-8"?>
  <!DOCTYPE properties SYSTEM "http://java.sun.com/dtd/properties.dtd">
   <properties>
   <comment>Configuration for English language tagging</comment>
   <entry key="tagger">http://medialab.di.unipi.it/cgi-bin/tagger?</entry>
   <entry key="language">en</entry>

<!-- Dependency Relations -->
